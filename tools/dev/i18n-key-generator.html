什么是国际化 (I18n)？

国际化（Internationalization，通常简写为 I18n，因为中间有 18 个字母）是软件开发中的一个重要过程，旨在设计和开发能够适应不同语言和地区的应用程序。在前端开发中，这通常意味着将硬编码的字符串提取到单独的资源文件中（如 JSON），并使用唯一的“键名”（Key）来引用它们。

为什么需要国际化键名生成器？

在大型项目中，手动为成百上千个文案创建键名是一项极其枯燥且容易出错的任务。开发者往往需要考虑：

  • 一致性： 确保所有键名遵循相同的命名规范（如驼峰式或下划线式）。
  • 唯一性： 避免不同模块使用相同的键名导致文案覆盖。
  • 可读性： 键名应当能直观反映文案的含义。

本工具通过智能算法，自动将中文或英文文案转换为标准化的键名。对于中文，它会自动提取拼音并进行格式化，极大地节省了开发时间。

国际化键名命名最佳实践

1. 模块化前缀： 使用层级化的命名方式，例如 login.button.submit，这有助于组织 JSON 文件并避免冲突。

2. 选择合适的格式：

  • camelCase： 现代前端框架（React/Vue）最常用的格式。
  • snake_case： 常见于后端或某些传统项目的配置。
  • kebab-case： 适合作为 CSS 类名或 HTML 属性。

3. 保持语义化： 键名应当描述“它是什么”而不是“它显示什么”。例如，使用 auth.error_message 而不是 auth.please_input_password。

如何在项目中使用生成的 JSON？

以 react-i18next 为例，你可以将生成的 JSON 直接放入你的语言包文件中：

// en.json
{
  "common": {
    "save": "Save",
    "cancel": "Cancel"
  }
}

// 在组件中使用
const { t } = useTranslation();
return <button>{t('common.save')}</button>;
