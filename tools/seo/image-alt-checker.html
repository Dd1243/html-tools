什么是图片 Alt 属性？

Alt（Alternative text）是 <img> 的替代文本。图片加载失败、用户使用屏幕阅读器，或搜索引擎抓取图片时，都会依赖它理解图片含义。对 SEO 而言，Alt 是图片进入图片搜索结果的重要信号；对无障碍（a11y）而言，它是合规与体验的基础。

本工具做什么？

把页面 HTML 粘贴进来后，工具会解析所有 img 标签，统计：

  • 正常 Alt：存在且非空，可读描述
  • 空 Alt：alt=""，常用于装饰性图片
  • 缺失 Alt：完全没有 alt 属性（优先修复）

同时给出 SEO 完整度评分，并支持按状态筛选图片列表，方便对照修复。

写作规范：好的 Alt 长什么样？

  • 描述内容，而不是文件名：写「红色跑鞋侧视图」，不要写「IMG_1234」。
  • 简洁准确：一般 5–15 个中文词/短语足够，避免段落堆砌。
  • 自然含关键词：可融入页面主题词，但禁止关键词堆砌。
  • 装饰图用空 Alt：纯分割线、背景纹理等无信息图片用 alt=""。
  • 功能图要说结果：按钮图标写「提交订单」，不要只写「图标」。

常见错误

  1. 大量图片缺失 alt，尤其是 CMS 上传后忘记填写
  2. 所有图片都写同一句「产品图片」
  3. 把整段广告文案塞进 alt
  4. 把重要信息只放在图里、正文不重复（无障碍与 SEO 双输）
  5. 把装饰图写成很长描述，屏幕阅读器噪音大

推荐工作流

  1. 浏览器打开目标页 → 查看源代码 / 导出 HTML 片段
  2. 粘贴到本工具，先处理「缺失 Alt」
  3. 再审查「空 Alt」是否真为装饰图
  4. 对内容图补写准确描述，回填 CMS 或代码
  5. 上线后用搜索控制台关注图片展现与可访问性抽检

与 Core Web / 无障碍的关系

Alt 本身不直接等于 LCP 分数，但图片优化常与尺寸、懒加载、格式（WebP/AVIF）一起做。无障碍审计（如 WCAG）会检查有意义的替代文本。本工具聚焦「有没有写好 Alt」，不替代完整可访问性审计。

能力边界

  • 不爬取整站，需你提供 HTML
  • 不自动改写线上代码，只做检测与评分
  • 不判断 alt 文案是否“足够好”，只区分有/空/无
  • 分析在浏览器本地完成，默认不上传你的源码
