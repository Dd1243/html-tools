端口号基础指南：网络通信的“门牌号”

在 TCP/IP 协议中，端口号（Port Number）是一个 16 位的数字，范围从 0 到 65535。如果说 IP 地址标识的是网络上的一台设备，那么端口号则标识了该设备上运行的特定应用程序或服务。

端口号的分类与范围

  • 知名端口 (0 - 1023)： 由 IANA 分配给特定的核心服务，如 80 (HTTP)、443 (HTTPS)、22 (SSH)。通常需要系统管理员权限才能监听。
  • 注册端口 (1024 - 49151)： 分配给特定的用户进程或应用程序，如 3306 (MySQL)、6379 (Redis)。
  • 动态/私有端口 (49152 - 65535)： 通常作为客户端连接服务器时的临时源端口。

如何利用端口号解决实际问题？

场景一：排查网站无法访问。 当你的网站打不开时，首先应检查服务器的 80 或 443 端口是否处于 LISTEN 状态。如果端口已监听但仍无法访问，可能是防火墙（如 iptables 或安全组）未放行。本工具能帮你快速确认这些服务的标准端口。

场景二：提升服务器安全性。 黑客通常会扫描常见端口。通过本工具了解每个端口对应的服务后，你可以关闭所有不必要的端口（如关闭不用的 21 FTP 端口），或者将敏感服务（如 SSH）更改到非标准的高位端口（如 2222），以降低被扫描到的几率。

场景三：配置局域网端口转发。 在进行 NAS 远程访问或内网穿透时，你需要准确知道服务的内网端口。例如，远程桌面 RDP 默认使用 3389，你可以通过路由器将其映射到外网的一个随机端口。

常见协议解析

TCP (传输控制协议)： 面向连接，提供可靠的数据传输，适用于 Web、邮件和文件传输。

UDP (用户数据报协议)： 无连接，速度快但不可靠，适用于视频直播、在线游戏和 DNS 查询。

安全建议

永远遵循“最小暴露原则”。除了必须对外提供服务的 80/443 端口外，其他如数据库端口（3306/5432）应严格限制仅内网访问或通过 VPN 连接。使用本工具定期核对你的开放端口清单，是维护网络安全的重要一环。
