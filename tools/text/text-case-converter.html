为什么要转换文本大小写？

在文字写作和程序开发中，不同场景对文本格式有着严格的要求。例如，英文标题通常需要单词首字母大写，而变量命名在不同的编程语言中则遵循不同的规范（如 Java 常用驼峰式，Python 常用蛇方式）。手动转换这些格式不仅繁琐，而且容易出错。

常见的命名规范与转换模式

  • 驼峰命名 (camelCase)： 单词首字母小写，后续单词首字母大写。常用于 JavaScript、Java、C# 中的变量名和函数名。
  • 帕斯卡命名 (PascalCase)： 所有单词首字母均大写。常用于类名、构造函数以及 C# 中的公共属性。
  • 蛇形命名 (snake_case)： 单词全部小写，并用下划线连接。是 Python、C 语言以及数据库字段命名的首选。
  • 短横线命名 (kebab-case)： 又称连字符命名。常用于 HTML 属性、CSS 类名以及 URL 路径。
  • 常量模式 (CONSTANT_CASE)： 单词全部大写，用下划线连接。通常用于定义全局常量或环境变量。

如何使用本工具？

1. 粘贴文本： 将原始文本粘贴到左侧输入框。工具会自动识别单词边界，无论您的输入是空格分隔、下划线分隔还是中线分隔。

2. 选择模式： 点击下方的按钮切换转换模式。所有的转换都是 实时进行 的，您可以立即在右侧看到效果。

3. 一键复制： 转换完成后，点击“复制转换结果”按钮，即可将其应用到您的代码或文档中。
