什么是测试假数据（Mock Data）？

在软件开发生命周期中，测试假数据是指为了测试、演示或模拟生产环境而人工生成的非真实信息。这些数据在结构上与真实数据完全一致，但内容是随机生成的。本工具通过复杂的算法模拟真实的命名习惯、地理规律及编码逻辑，为您提供高质量的测试素材。

为什么开发过程中需要假数据生成器？

直接使用生产环境的真实用户数据进行开发和测试不仅违法法律法规（如 GDPR 或《数据安全法》），还存在极大的安全风险。本工具可以解决以下核心痛点：

  • 保护隐私合规： 完全合法的随机数据，让开发团队无需接触敏感的真实用户信息。
  • 快速填充数据库： 无需手动输入，一键生成成百上千条记录，用于压力测试或分页功能校验。
  • 前端 UI 展示： 在页面设计阶段，使用真实感的文字长度和内容替换单调的“测试测试”，能更准确地预判布局效果。
  • 跨平台导出： 支持 JSON（用于前端）、CSV（用于 Excel/数据分析）、SQL（用于数据库导入）等多种行业标准格式。

如何正确使用生成的数据？

  1. 选择字段： 根据您的数据库表结构或 API 需求，勾选所需的字段（如只需要姓名和手机号）。
  2. 设定场景： 针对国内业务选择“中文”，针对出海项目选择“英文”，确保数据的本土化特征。
  3. 选择格式： 开发联调推荐使用 JSON；批量导入 MySQL/PostgreSQL 推荐使用 SQL 格式。

常见问题解答 (FAQ)

Q: 生成的手机号和身份证号是真的吗？
A: 不是。虽然它们符合号码段规则和校验和算法（如身份证的模 11 校验），但它们并没有在任何政府系统中注册，无法拨打或用于实名认证。

Q: 为什么生成的地址看起来有些奇怪？
A: 算法通过随机组合省市区和街道名生成地址。虽然每个地名都是真实的，但组合在一起可能并不代表一个物理存在的真实位置。

Q: 生成数据的数量有限制吗？
A: 为了保证浏览器性能，本在线版工具单次最多支持生成 100 条数据。如需更大规模数据，建议分批生成或联系我们获取离线版。
