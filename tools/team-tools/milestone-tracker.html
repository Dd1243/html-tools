里程碑追踪器完整使用指南

跟踪项目关键节点：项目名 #projName、目标结束 #projEnd，列表管理里程碑标题与日期，可标记完成、删除。总览进度条 #overallBar，统计总数/已完成/逾期等。数据键 milestone-data；支持分享状态、导出报告、清空。主题存 theme。适合立项后的节点看板，不是甘特依赖网络，也不自动发提醒邮件。

操作与状态

  • addMilestone 添加；toggleDone 完成切换；delM 删除。
  • updateUI 刷新列表、进度与统计（含逾期判断相对今天/项目结束）。
  • saveState/loadState 读写 localStorage。
  • shareState 生成可分享快照；exportReport 导出文本报告。
  • clearAll 清空前请先导出。

管理建议

  1. 里程碑 5–12 个为宜，过多变任务清单应拆到任务工具。
  2. 日期用真实验收点，完成定义写在标题括号内。
  3. 周会看逾期统计，先处理红项。
  4. 项目结束日帮助判断整体是否漂期。
  5. 导出报告贴进周报；分享链接给干系人只读对齐。

边界

  • 无依赖前置、无资源分配、无基线对比高级分析。
  • 逾期算法基于本地日期，时区以设备为准。
  • 分享无权限控制，注意脱敏。
  • 不推送通知，需人工打开页面。

常见问题

刷新后数据在吗？

在，键 milestone-data。

和甘特图怎么选？

甘特看条形时间并行；里程碑看节点完成与逾期。

完成能取消吗？

再 toggle 即可取消完成（以 UI 为准）。

进度条如何计算？

通常按已完成数/总数，具体以 updateUI 为准。

与站会/WBS 配合

WBS 拆叶子任务，里程碑收关键验收点，站会同步阻塞。不要把所有 todo 塞进里程碑。公共电脑导出后清空。主题切换不影响数据。

逾期与进度解读

逾期统计帮助暴露风险，但要区分「日期填错」与「真延期」。完成打勾应在验收通过后，而不是「差不多了」。进度条偏完成计数，不代表工作量加权。项目结束日用于整体紧迫感，可与公司里程碑日历对照。

分享与导出

shareState 适合同步快照；exportReport 适合周报粘贴。链接无权限，勿含未公开融资节点。键 milestone-data 本机；换电脑先导出。与甘特、WBS 分工：节点 / 条形 / 分解树。

本地数据与使用边界

本页草稿与结果主要保存在当前浏览器环境，清除站点数据、更换设备或隐私模式可能导致丢失，重要报价、名片、议程、投票与表单请及时导出、复制或打印备份。公共电脑使用完毕后应清空敏感信息。工具用于提升效率，不替代正式商务、项目管理或合规流程，请以组织制度与法规为准。

周会怎么用里程碑

会前打开页面看逾期与未完成；会中只讨论红项原因与下一步主人。完成勾选必须对应可验收产出。项目名与结束日写在顶部，避免多项目混在一个列表。导出报告贴周报，分享链接给远程干系人。键 milestone-data；清空前先 export。

和甘特/任务的边界

里程碑是「节点是否达成」；甘特是时间条；任务跟踪是日常勾选。不要把几十条琐事塞进里程碑。主题 theme 与业务数据分离。设备日期不准会导致逾期误报，请校准系统时间。
