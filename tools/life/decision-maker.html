决策助手完整使用指南

决策助手提供四种随机决策玩法：转盘、抛硬币、掷骰子、随机选择。转盘默认三项「选项1/2/3」，可继续添加并在 canvas 上绘制扇区后旋转；硬币给出正面/反面动画；骰子可选 1–3 个，用 Unicode 骰子字符显示；随机模式从你自定义列表里抽取。历史记录区会追加每次结果，便于回看。主题按钮可切换浅色/深色。它适合打破“小选择拖延”，不适合重大法律/医疗/财务决策。

四个模式分别做什么

  • 转盘：输入选项点 ➕ 或 Enter 添加；点「开始转动」旋转 canvas 转盘并展示结果。
  • 抛硬币：点「抛硬币」，硬币翻转动画后给出正/反结果。
  • 掷骰子：先选 1/2/3 个骰子，再掷出；结果用 ⚀–⚅ 显示。
  • 随机：维护选项列表后点「随机选择」抽取一项。
  • 底部历史：记录模式与结果文本，方便对比多次尝试。

推荐用法

  1. 先把“真正可接受”的选项写全，再开始随机，避免抽到不可执行项。
  2. 二选一用硬币；多选项用转盘或随机列表。
  3. 需要一点仪式感时用转盘动画，需要数字结果时用骰子。
  4. 重大决策请先做利弊表，把本工具仅作破冰，不作为唯一依据。

适用场景

  • 午饭吃什么、周末去哪、看哪部影片
  • 团队破冰小游戏、抽签决定发言顺序
  • 在多个都可行的方案里快速推进

边界

  • 结果由前端伪随机产生，不具备密码学随机强度。
  • 不是加权决策：默认各选项等权（除非页面另有权重实现）。
  • 历史记录在页面会话中展示，刷新后通常不作为长期档案。
  • 不能替代正式投票系统与可审计抽签流程。

常见问题

转盘没选项能转吗？

默认有三个示例选项；若删到为空，请先添加至少一项再转。

骰子最多几个？

界面提供 1/2/3 个切换，不是任意数量。

能给选项设置不同概率吗？

当前是娱乐向等权随机，不做复杂权重配置面板。

结果可复现吗？

一般不可复现。若要可审计抽签，请使用带种子与公示规则的专业流程。

适合做公司制度决策吗？

不适合。请用于低风险个人/轻松场景，重要事项走会议与数据评估。

主题切换影响结果吗？

不影响随机逻辑，只改变界面明暗样式。

模式切换与控件对照

四个区块由 showSection 切换：转盘 #wheelSection、硬币 #coinSection、骰子 #diceSection、随机列表 #randomSection。转盘：选项列表 #wheelOptions，输入 #wheelInput，画布 #wheelCanvas，结果 #wheelResult；增删用 addWheelOption / removeWheelOption，绘制 drawWheel，开转 spinWheel。硬币：flipCoin 驱动 #coinInner 动画，结果进 #coinResult。骰子：setDiceCount 配合 #dice1/#dice2/#dice3，容器 #diceContainer，掷出 rollDice，结果 #diceResult。随机：列表 #randomOptions、输入 #randomInput、pickRandom 写入 #randomResult。每次有效结果经 addHistory 追加到 #historyList。

让随机更「可用」的用法

先过滤不可执行选项再开转，例如午餐列表只保留「今天真能去」的店。二选一用硬币最快；三选以上用转盘或随机列表。骰子适合需要数字分边或桌游辅助，不是自定义文案池。历史区用来对比「连抽是否总偏向某一项」——若你主观不适，应回到利弊分析，而不是把伪随机当命运。主题 toggleTheme 只改明暗，不改概率模型。

边界再强调

结果来自浏览器伪随机，无密码学强度，也无权重面板。历史以当前页展示为主，刷新后通常不作为长期档案。公司制度、抽奖合规、医疗财务等场景请走可审计流程。空选项时不要强转：转盘至少保留一项；随机列表同理先 addRandomOption 再抽。
