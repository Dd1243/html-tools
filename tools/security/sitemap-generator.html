什么是 XML Sitemap？为什么它对 SEO 至关重要？

XML Sitemap（站点地图）是一个列出了网站上所有重要页面 URL 的文件。它的主要作用是“主动告知”搜索引擎：我的网站有哪些内容，哪些页面是最近更新的。虽然爬虫可以通过链接自动发现页面，但 Sitemap 能够极大提升抓取效率，特别是对于新站或结构复杂的巨型网站。

关键参数解析

  • loc: 页面的完整 URL。
  • lastmod: 页面最后修改的时间，告诉爬虫是否需要重新抓取。
  • changefreq: 页面可能的更新频率（如 daily, weekly）。
  • priority: 页面在站内的相对重要程度（0.0 到 1.0）。

解决“新页面不收录”的痛点

很多开发者发现新写的文章在 Google 中几天甚至几周都没有排名。这往往是因为爬虫还没“逛”到那里。通过生成 Sitemap 并提交给 Google Search Console ，您可以：

  • 加速收录： 缩短从发布到被索引的时间窗口。
  • 优化抓取配额： 让爬虫优先关注高价值页面，而不是浪费在无关紧要的附件页上。
  • 监控抓取状态： 在 Search Console 中查看哪些页面已收录，哪些存在报错。

XML Sitemap 的最佳实践

  • 文件大小限制： 单个 Sitemap 文件不应超过 50,000 个 URL 或 50MB（未压缩）。
  • 使用规范链接： 仅列出您希望搜索引擎收录的 Canonical URL，不要包含重复内容。
  • 排除无价值页面： 像后台登录页、个人中心、购物车等页面不应出现在 Sitemap 中。

如何提交您的 Sitemap？

  1. 生成文件： 使用本工具生成 XML 内容并保存。
  2. 上传： 将 sitemap.xml 上传到网站根目录。
  3. 告知搜索引擎：
    • 在 Google Search Console 的“站点地图”板块提交 URL。
    • 在 robots.txt 中加入 Sitemap: https://yourdomain.com/sitemap.xml 。
