在线甘特图制作工具

添加任务名称与起止日期，按天/按周可视化进度条。数据保存在本地浏览器，可导出图片、打印或复制分享链接。

项目任务

怎么用

  • 填写任务名与起止日期后点添加
  • 短项目用按天，跨多周用按周视图
  • 悬停任务名可删除单行
  • 分享链接会把任务写入 URL hash
  • 导出图片适合贴进周报

常见问题

Q: 刷新后任务还在吗？
A: 有 hash 优先用 hash，否则读 localStorage 键 gantt_chart_data_v2。
Q: 结束日当天算工期吗？
A: 条宽对结束日 +1 天，按含首尾理解更贴近显示。
Q: 能算关键路径吗？
A: 不能。本页无依赖线与关键路径，定位是轻量时间条可视化。
Q: 数据会上传吗？
A: 不会。本地存储 + 可选链接分享，不经过服务器。

在线甘特图制作工具完整使用指南

本页用任务数组 tasks 渲染简易甘特图：每项含名称、开始日期、结束日期。添加区填写 #taskName、#startDate、#endDate 后点「添加任务」；时间轴支持 #viewScale 的「按天 / 按周」视图。任务条按调色板 COLORS 循环着色，左侧列表可删除单行（deleteTask）。数据写入 localStorage 键 gantt_chart_data_v2，同时 updateUrl 把任务 JSON 经 Base64 放进 URL hash 便于分享。导出高清图依赖 html2canvas（#exportBtn），打印用 #printBtn，清空用 #clearBtn。适合轻量排期可视化，不是微软 Project 级依赖/资源管理。

渲染与日期规则

  • getRange()：取所有任务最早开始与最晚结束，并左右留白（开始前约 2 天、结束后约 5 天）。
  • 时间头按天生成单元格，周末加 weekend 样式；按周视图会合并/隐藏非周一列。
  • 任务条 left/width 用「天数 × 单元格宽度 35px」估算，时长含结束日（+1 天）。
  • 左侧任务名列表与右侧时间条同步行高。
  • save()：写 localStorage、重绘、更新 hash；加载优先 hash，否则本地缓存。

推荐使用步骤

  1. 先列 5–15 条关键任务，日期用真实起止，避免全挤在同一天。
  2. 短项目用按天视图；跨多周项目切换按周看节奏。
  3. 调整后点「分享链接」复制地址，对方打开即可还原任务。
  4. 需要贴进周报时用「导出高清图片」；纸质材料用打印。
  5. 清空前确认：清空会抹掉当前列表并同步 hash/本地存储。

能力边界

  • 无前置任务依赖线、无关键路径计算、无资源负荷与进度百分比。
  • 大量任务或超长跨度会使时间轴很宽，浏览器可能卡顿。
  • hash 分享不含权限控制，勿放入机密项目细节。
  • html2canvas 导出效果受 DOM/字体影响，复杂样式可能略有偏差。
  • 无 Excel 导入；需逐条添加或自行改本地数据。

常见问题补充

刷新后任务还在吗？

有 hash 用 hash；否则读 gantt_chart_data_v2。清浏览器缓存会丢本地副本。

结束日当天算不算工期？

条宽计算对结束日做了 +1 天处理，按「含首尾」理解更贴近显示。

和项目时间线如何分工？

甘特看时间条并行关系；时间线页更适合里程碑节点与状态勾选。两者可配合：时间线对外汇报，甘特对内排期。

排期建模建议

先写里程碑级任务（需求冻结、开发、联调、上线），再拆并行条。开始/结束用真实日历日，避免用同一天表示整周工作导致条过短。颜色按添加顺序循环调色板，仅作视觉区分，不代表优先级。周末单元格有样式提示，但不会自动跳过工期——是否含周末由你填的日期决定。分享前检查 hash 是否过长：任务极多时 URL 可能不便分享，可改为导出图片。

本地与链接双通道

键名 gantt_chart_data_v2 存当前浏览器；updateUrl 同步 hash。打开带 hash 的链接会优先解析快照，适合评审会投屏。注意：hash 可被转发，勿写入薪资、未公开客户名等敏感字段。导出高清依赖 html2canvas CDN，若按钮无响应请检查网络是否加载到脚本。

如何用甘特图把项目排期说清楚？

甘特图用横向时间条表达「谁在什么时候做什么」。相比纯文字计划，它能立刻看出并行任务、工期重叠与空档，适合周会、迭代评审与跨团队对齐。

1. 先结果后任务

先定交付结果（上线、验收、冻结），再拆可执行任务条。条不宜过碎：5–15 条通常足够支撑一次会议讨论。

2. 日期要可核对

起止日期应来自真实日历，而不是「大概两周」。本工具按你填写的日期画条，周末仅作视觉提示，不会自动跳过非工作日。

3. 对外与对内分层

对外可用导出图片展示关键并行关系；对内再细化到任务跟踪清单。需要节点状态勾选时，可配合站内项目时间线工具。

4. 不要用轻量甘特硬扛专业需求

关键路径、资源冲突、基线对比、进度自动汇总等，请使用专业项目管理软件。本页定位是「五分钟画出时间条，让会议对齐」。
