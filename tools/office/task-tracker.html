在线任务跟踪器

科学管理待办事项，自定义优先级，实时追踪项目进度。数据 100% 存储于您的浏览器本地。

怎么用

  • 输入任务并选择优先级后添加
  • 勾选完成，统计数字同步
  • 用筛选看全部/进行中/已完成
  • 清理已完成保持列表干净
  • 数据在本地键 task_tracker_v2_data

常见问题

Q: 和清单制作有何不同？
A: 任务跟踪强调优先级与状态筛选；清单更适合一次性检查勾选与拖拽顺序。
Q: 刷新后任务还在吗？
A: 在。使用 localStorage 键 task_tracker_v2_data。
Q: 支持多人协作吗？
A: 不支持实时协同，仅本机；需要协作请导出或另用协同工具。
Q: 数据会上传吗？
A: 不会。

在线任务跟踪器完整使用指南

轻量待办跟踪：输入任务 #taskInput，优先级 #prioritySelect（高/中/低），提交到列表。统计总任务、未完成、已完成数量；可用 #statusFilter 筛选。每条可完成勾选、删除；支持清除已完成与清空全部。数据键 task_tracker_v2_data。优先级以色条与中文标签展示。适合个人执行清单与小团队公开看板投影，不是完整 Jira/权限工作流。

数据字段与函数

  • 任务对象含 id（generateId）、文本、priority、完成状态、时间（formatDate）等。
  • addTask 入队；toggleTask 切换完成；deleteTask 删除。
  • renderTasks 按筛选渲染；loadTasks/saveTasks 读写 localStorage。
  • 优先级文案映射：high 高、medium 中、low 低。
  • 清除已完成只删 done；清空全部需确认（以 UI 为准）。

推荐用法

  1. 早间写下今日 3–5 条，高优先级不超过 2 条，避免全是「高」。
  2. 执行时用筛选看 active；收工清 completed 保持列表干净。
  3. 任务名写「动词+对象+完成标准」，如「提交报销单（含发票 PDF）」。
  4. 与站会计时配合：站会只报阻塞，细节勾在本页。
  5. 公共电脑用完清空，避免暴露工作内容。

边界

  • 无经办人字段、无截止日期提醒推送、无附件。
  • 无多人实时同步；同浏览器多标签可能互相覆盖写入。
  • 筛选不改变底层数据，只改变视图。
  • 不替代公司正式任务系统审计要求。

常见问题

刷新后任务在吗？

在，键名 task_tracker_v2_data。

和清单 checklist 区别？

跟踪器强调优先级与状态统计；清单强调勾选进度条与拖拽排序/分享。

和待办 todo-list 页？

同属待办家族但存储键与 UI 不同，不要假设数据互通。

误删能恢复吗？

一般不能，重要事项请同时写在正式系统。

优先级纪律

若所有任务都标高，优先级失效。可规定：高=今天必须；中=本周；低=有空再做。每天重新整理，完成即勾选，避免列表变成焦虑墙。投影给团队时先过滤掉私人低优先级项。

筛选与日常节奏

早上看 active，晚上清 completed。过滤不删除数据，切换「全部」仍在。高优先级色条帮助扫视，但不要滥用。键 task_tracker_v2_data 仅本机；团队共享请用正式工具或各自列表。清除已完成可减小列表噪音；清空全部用于项目结束归档后。

命名与拆分

超过两小时的事先拆，或先在 WBS 分解再抄叶子到此。阻塞任务在文案前加【堵】便于站会扫。生成 id 与日期字段用于展示，不必手填。多标签页同时改可能丢写入，建议单标签操作。

隐私

任务可能含客户名，公共电脑用完 clearAll。投屏前先过滤私人项。本页无加密存储。

本地数据与使用边界

本页结果与草稿主要保存在当前浏览器环境中，清除站点数据、更换设备或使用隐私模式可能导致内容丢失，重要清单、密文、任务与 WBS 请及时导出、复制或截图备份。公共电脑使用完毕后应清空页面上的敏感输入，避免姓名、口令、项目结构等留给下一位用户。工具用于提升个人与小团队效率，不替代组织内正式的项目管理、考勤人事、密钥托管或合规审计系统；请以公司制度与当地法规为准。
