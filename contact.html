欢迎就故障、内容错误、隐私与广告问题，或新工具建议联系 WebUtils。我们优先处理影响访问、安全与核心功能可用性的反馈。

主要反馈渠道

GitHub Issues（推荐）

用于报告 Bug、提出功能建议、讨论分类与文档改进。请新建 Issue，并尽量按下方模板补充信息：

https://github.com/Dd1243/html-tools/issues

项目仓库

查看源码、目录结构与历史变更；也可通过仓库入口导航到 Issues。

https://github.com/Dd1243/html-tools

目前站点以 GitHub 作为公开协作与问题追踪入口。若您无法使用 GitHub，可在 Issue 中说明限制，我们会在后续评估补充其他联系方式。

请尽量提供这些信息

完整信息能显著缩短定位时间。建议在反馈中包含：

  1. 页面地址：完整 URL，例如 https://essays4u.net/tools/seo/robots-generator
  2. 问题现象：期望结果 vs 实际结果；是否必现。
  3. 复现步骤：输入了什么、点了哪个按钮、第几步出错。
  4. 环境信息：浏览器（Chrome / Edge / Safari / Firefox 等）与版本、桌面或手机、操作系统。
  5. 截图或报错：若有控制台报错、空白页或布局错乱，请附截图（注意打码敏感内容）。

我们接受哪些反馈

  • 工具计算结果错误、转换失败、按钮无响应、复制/导出异常。
  • 页面打不开、样式错乱、移动端无法操作、内链失效。
  • 文案错误、误导说明、重复页面、乱码或分类放错。
  • 隐私政策、Cookie、广告展示、第三方资源相关疑问。
  • 新工具建议、现有工具增强建议、文档与示例补充建议。
  • 疑似侵权、违法违规内容或滥用报告（请尽量提供链接与证据）。

处理优先级与回复预期

  • P0：全站无法访问、核心工具大面积失效、明显安全/隐私风险。
  • P1：单个常用工具结果错误、关键页面布局损坏、错误跳转。
  • P2：文案优化、体验建议、新工具需求、非阻塞型改进。

我们会尽力及时查看公开 Issue，但 WebUtils 是持续维护的工具集合， 不承诺固定的 SLA 响应时限。紧急程度越高、信息越完整，越容易被优先处理。功能建议可能会合并、延期或暂不实现。

隐私、广告与数据相关请求

若您的问题涉及 Cookie、Google 广告、第三方脚本或本站如何处理数据，请先阅读 隐私政策。您也可以通过 Google 广告设置 管理个性化广告。

反馈隐私相关问题时，请不要在公开 Issue 中粘贴密码、证件号、完整密钥、未脱敏日志或其他敏感数据。如需举例，请使用虚构样本或打码后的片段。

商务、合作与广告

WebUtils 以提供免费工具为主，并通过广告等方式支持维护。若您有合作意向、内容纠错或广告相关沟通，请通过上述 GitHub Issues 提交，标题请写明「合作 / 纠错 / 广告」以便分类。我们不保证接受所有合作请求。
